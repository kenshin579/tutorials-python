# 벡터 저장소 비교

## 벡터 저장소란?

벡터 저장소(Vector Store)는 텍스트를 고차원 벡터로 변환한 임베딩을 저장하고, 유사도 기반 검색을 수행하는 전문 데이터베이스다. RAG 시스템의 핵심 인프라 컴포넌트다.

## 주요 벡터 저장소

### ChromaDB

오픈소스 임베딩 데이터베이스로, 설치와 사용이 간단하다. Python 패키지로 설치하면 별도 서버 없이 임베디드 모드로 바로 사용할 수 있다. 프로토타이핑과 소규모 프로젝트에 적합하다.

### FAISS (Facebook AI Similarity Search)

Meta(Facebook)가 개발한 고성능 벡터 검색 라이브러리다. 다양한 인덱싱 알고리즘(Flat, IVF, HNSW, PQ)을 지원하며, GPU 가속도 지원한다. 연구 및 고성능이 필요한 환경에 적합하다.

### Pinecone

완전 관리형 벡터 데이터베이스 서비스다. 서버리스 아키텍처로 자동 확장되며, 인프라 관리 부담이 없다. 프로덕션 환경과 엔터프라이즈에 적합하다.

### Weaviate

오픈소스 벡터 데이터베이스로, 벡터 검색과 BM25 키워드 검색을 결합한 하이브리드 검색을 기본 지원한다. GraphQL API를 제공하며, 멀티모달 데이터도 처리할 수 있다.

## 인덱싱 알고리즘

### Flat (Brute-force)
모든 벡터와 거리를 계산하는 방식. 100% 정확하지만 느리다. 10만 건 미만의 소규모 데이터에 적합하다.

### IVF (Inverted File Index)
k-means로 벡터 공간을 클러스터로 분할하고, 검색 시 가까운 클러스터만 탐색한다. 대규모 데이터에서 효율적이다.

### HNSW (Hierarchical Navigable Small World)
계층적 그래프 구조를 사용하여 탐색한다. 높은 정확도와 빠른 검색 속도를 동시에 제공하며, 현재 가장 많이 사용되는 알고리즘이다.

## 선택 가이드

- 빠른 프로토타이핑 → ChromaDB
- 연구/실험, 최대 성능 제어 → FAISS
- 프로덕션, 관리 부담 최소화 → Pinecone
- 하이브리드 검색 필요 → Weaviate
